### A3. 만족도 데이터 활용 (어드민 운영 범위)

**결정**: 선택지 2 — 목록 + 디테일 모달 기본형

**답변**: 2번으로 하되, 처리완료는 업무에서 필요없을 것으로 보임

**포함 항목**:
- `/admin/feedback` 목록 페이지
- 클릭 시 디테일 모달 (질문/답변 전문 + 점수 + 사유 + 자유 의견)

**제외 항목**:
- 처리완료 토글 (불필요)
- 통계 대시보드 (추후)

**개발 영향**: 연구소 어드민 작업 범위에 반영 필요 (`10_연구소_전달용_API요청서` 업데이트).

---

### A4. 부정 평가 사유 칩 분기

**결정**: 선택지 1 — 점수별 칩 분기

**답변**: 선택지 1

**칩 구성**:

| 점수 | 칩 풀 |
|---|---|
| 4~5점 (긍정) | 정확해요 / 도움이 됐어요 / 빨랐어요 / 이해하기 쉬워요 / 기타 |
| 1~3점 (부정) | 부정확해요 / 내용이 부족해요 / 느렸어요 / 이해하기 어려워요 / 기타 |

**개발 영향**:
- `FeedbackSurveyData.reasons` 단일 풀 → `positiveReasons` / `negativeReasons` 분리 또는 score 기반 런타임 분기
- `FeedbackSurvey.tsx`: 선택된 score가 4~5이면 긍정칩, 1~3이면 부정칩 렌더
- `mockScenes.ts`: 두 칩 풀 데이터 추가

### A6. 자유 의견 보관 정책

**결정**: 미정

**답변**: 현재 해당 정책 없음으로 미정

**후속 과제**: 채팅 데이터 보관 정책 수립 시 만족도 데이터(자유 의견) 보관 룰 동시 확정 필요.

**현재 처리**: 정책 확정 전까지는 자유 의견 저장 구조만 만들고, 보관 기간·마스킹은 미적용 상태로 유지.

### A7. 위젯 노출 예외

**결정**: supervisor 프롬프트에서 걸러지는 질의 = 위젯 미노출 / 그 외 모든 정상 답변 = 노출

**답변**: supervisor 프롬프트에서 걸러지는 질문은 미노출

**개발 영향**:
- supervisor가 걸러내는 경우 응답 메시지에 별도 플래그 필요 (개발팀 확인)
- 해당 플래그가 있을 때 `feedback-survey` 블록 미포함 처리

**후속 확인**: supervisor 걸러지는 조건이 프론트 응답 데이터에서 식별 가능한지 개발팀 확인 필요.
